# 2130. 链表最大孪生和 (Maximum Twin Sum of a Linked List)

**难度**: Medium  
**链接**: https://leetcode.cn/problems/maximum-twin-sum-of-a-linked-list/  
**日期**: 2026-06-17  
**相关话题**: 栈、链表、双指针

## 题目描述

在一个大小为 `n` 且 `n` 为偶数的链表中，对于 `0 <= i <= (n / 2) - 1` 的 `i`，第 `i` 个节点（下标从 0 开始）的**孪生节点**为第 `(n-1-i)` 个节点。

- 比方说，`n = 4` 那么节点 0 是节点 3 的孪生节点，节点 1 是节点 2 的孪生节点。

**孪生和**定义为一个节点和它孪生节点两者值之和。

给你一个长度为偶数的链表的头节点 `head`，请你返回链表的**最大孪生和**。

### 示例

**示例 1**:
```
输入: head = [5,4,2,1]
输出: 6
解释: 
  节点 0 和节点 1 分别是节点 3 和 2 的孪生节点。孪生和都为 6。
  链表中没有其他孪生节点。
  所以，链表中的最大孪生和为 6。
```

**示例 2**:
```
输入: head = [4,2,2,3]
输出: 7
解释:
  链表中的孪生节点为:
  - 节点 0 和节点 3 的孪生和为 4 + 3 = 7
  - 节点 1 和节点 2 的孪生和为 2 + 2 = 4
  所以，最大孪生和为 max(7, 4) = 7
```

**示例 3**:
```
输入: head = [1,100000]
输出: 100001
解释:
  链表中只有一对孪生节点，孪生和为 1 + 100000 = 100001
```

### 约束
- 链表的节点数目是 `[2, 10^5]` 中的**偶数**
- `1 <= Node.val <= 10^5`

---

## 解题思路

### 核心洞察

孪生节点对 `(i, n-1-i)` 关于链表中心对称。因此，只需要：
1. 将链表从中点分成两半
2. 反转后半部分
3. 同时遍历两半计算孪生和

### 算法: 快慢指针 + 反转链表

**步骤 1: 找到中点（快慢指针）**
- `slow` 每次走 1 步，`fast` 每次走 2 步
- `fast` 到末尾时，`slow` 正好在后半部分的起点

**步骤 2: 反转后半部分链表**
- 使用标准的 `prev/curr/next` 三指针反转法
- 反转后，后半部分变为从尾到头

**步骤 3: 计算最大孪生和**
- 同时遍历前半部分（从头开始）和反转后的后半部分
- `twinSum = firstHalf.val + secondHalf.val`
- 更新最大值

## 复杂度分析

| 复杂度 | 值 | 说明 |
|--------|-----|------|
| **时间复杂度** | **O(n)** | 遍历链表三次：找中点 O(n/2)、反转 O(n/2)、计算和 O(n/2) |
| **空间复杂度** | **O(1)** | 只使用常数个指针变量，没有使用额外数据结构 |

### 方法签名
```java
public int pairSum(ListNode head)
```

### 关键点
1. 使用快慢指针精确找到中点（偶数长度时 slow 指向后半部分第一个节点）
2. 反转链表时注意保存 `next` 指针，防止断链
3. 遍历计算时以反转后的后半部分为循环条件（因为它的长度可能略短）
